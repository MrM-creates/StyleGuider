# Recommendation Seeds

Diese Seed-Dateien beschreiben Beispiel-Eingaben für die spätere Recommendation-Logik von StyleGuider.

Jeder Seed enthält:

- ein kompaktes Markenprofil,
- Kommunikationskanäle,
- gewünschte Wirkung,
- Designpräferenzen,
- und eine erwartete Stil-Empfehlung.

## Zweck

Die Seeds dienen dazu,

- die Designachsen zu testen,
- Stilfamilien gegeneinander abzugrenzen,
- Recommendation-Heuristiken zu prüfen,
- und spätere Ranking-/Scoring-Logik nachvollziehbar zu machen.

## Hinweis

Die erwarteten Empfehlungen sind bewusst als **Primär- und Sekundärfit** formuliert.
Das ist realistischer als eine künstlich starre "eine Marke = genau ein Stil"-Logik.
